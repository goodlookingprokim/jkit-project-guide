# AXYS 사용 안내서

> AI 에이전트와 함께하는 학교 업무 혁신

## AXYS란 무엇인가?

AXYS(AI School Agent System)는 학교 조직의 구성원(교장, 교감, 부장교사 등)을 AI 에이전트로 정의하여, AI가 학교 업무를 도와주는 시스템입니다.

쉽게 말해, **학교에서 일어나는 다양한 업무를 AI 팀원들이 함께 도와주는 도구**입니다.

### 어떤 일을 할 수 있나요?

- **교육과정 편성**: 교무부장 AI가 편성 기준안을 작성하고, 교감 AI가 검토합니다
- **학교 행사 기획**: 교감 AI가 전체 기획을 총괄하고, 각 담당 AI가 세부 계획을 수립합니다
- **예산 편성**: 행정실장 AI가 예산안을 작성하고, 부서별 배정을 조율합니다
- **학생 생활지도**: 학년부장 AI가 지도 방침을 수립하고, 상담교사 AI가 전문 상담을 지원합니다
- 그 외 13개 학교 업무 시나리오를 지원합니다

### AI 에이전트 팀 구성

| 직책 | 역할 | 무엇을 도와주나요? |
|------|------|-------------------|
| 교장 | 최종 의사결정 | 중요한 결정의 최종 승인, 학교 비전 제시 |
| 교감 | 운영 총괄 | 업무 조율, 행사 기획, 위기 대응 |
| 교무부장 | 교육과정 | 시간표 편성, 학사일정, 성적 관리 |
| 연구부장 | 혁신/연수 | 교원 연수, 디지털 전환, 연구 사업 |
| 학년부장 | 학년 관리 | 학생 생활지도, 학년 행사, 학기 전환 |
| 담임교사 | 학급 경영 | 학부모 소통, 학생 관찰, 출결 관리 |
| 교과교사 | 수업 전문 | 교과 운영, 평가, 수업 연구 |
| 상담교사 | 학생 상담 | 심리 상담, 위기 개입, 학부모 상담 |
| 행정실장 | 예산/행정 | 예산 편성, 시설 관리, 물품 조달 |
| 학생대표 | 학생 의견 | 학생 자치, 의견 수렴, 행사 참여 |

## 설치 방법

### 준비물

- AI 도구 중 하나가 설치되어 있어야 합니다:
  - [Claude Code](https://claude.ai/code) (권장)
  - [Gemini CLI](https://github.com/google-gemini/gemini-cli)
  - [Codex CLI](https://github.com/openai/codex)
- 터미널(명령 프롬프트)을 열 수 있어야 합니다

### 설치 순서

1. AXYS 저장소를 다운로드합니다:

```bash
git clone https://github.com/your-org/axys.git
cd axys
```

2. 설치 스크립트를 실행합니다. `--school` 뒤에 학교명을, `--platform` 뒤에 사용할 AI 도구를 입력합니다:

```bash
# Claude Code를 사용하는 경우
./install.sh --school "우리학교" --platform claude-code

# Gemini CLI를 사용하는 경우
./install.sh --school "우리학교" --platform gemini-cli

# Codex CLI를 사용하는 경우
./install.sh --school "우리학교" --platform codex-cli

# 모든 플랫폼용 파일을 한번에 생성하는 경우
./install.sh --school "우리학교" --platform all
```

3. 생성된 파일을 프로젝트에 복사합니다:

```bash
# Claude Code의 경우 - 생성된 파일을 프로젝트 루트에 복사
cp -r output/claude-code/.claude /your/project/
cp output/claude-code/CLAUDE.md /your/project/

# Gemini CLI의 경우
cp output/gemini-cli/GEMINI.md /your/project/

# Codex CLI의 경우
cp output/codex-cli/AGENTS.md /your/project/
```

4. 설치가 완료되었습니다.

## Claude Code에서 사용하기

Claude Code는 AXYS를 가장 완전하게 지원하는 플랫폼입니다.

### 에이전트 호출하기

자연어로 요청하면 관련 에이전트가 자동으로 활성화됩니다. 특별한 명령어 없이 자연스럽게 대화하면 됩니다.

```
"교육과정 편성을 도와줘"
→ 교무부장 에이전트가 활성화되어 편성 절차를 안내합니다

"학교 행사 기획을 시작하고 싶어"
→ 교감 에이전트가 활성화되어 기획 프로세스를 진행합니다

"예산을 편성해야 해"
→ 행정실장 에이전트가 활성화되어 예산 작성을 지원합니다
```

### 시나리오 실행하기

슬래시 커맨드(`/`)를 사용하면 정해진 업무 시나리오를 단계별로 진행할 수 있습니다.

| 커맨드 | 시나리오 | 주관 |
|--------|----------|------|
| `/axys-curriculum-planning` | 교육과정 편성 | 교무부장 |
| `/axys-event-planning` | 학교 행사 기획 | 교감 |
| `/axys-budget-planning` | 예산 편성 | 행정실장 |
| `/axys-student-guidance` | 학생 생활지도 | 학년부장 |
| `/axys-teacher-development` | 교원 연수 계획 | 연구부장 |
| `/axys-safety-plan` | 학교 안전 계획 | 교감 |
| `/axys-evaluation` | 학교 자체 평가 | 교감 |
| `/axys-parent-communication` | 학부모 소통 | 교감 |
| `/axys-innovation-project` | 혁신학교/특색사업 | 연구부장 |
| `/axys-semester-transition` | 학기 전환 업무 | 교감 |
| `/axys-emergency-response` | 위기 대응 매뉴얼 | 교감 |
| `/axys-digital-transition` | 디지털 전환 | 연구부장 |
| `/axys-community-connection` | 지역사회 연계 | 연구부장 |

### 예시 대화 1: 교육과정 편성

```
사용자: /axys-curriculum-planning

교무부장(AI): 안녕하세요! 우리학교의 교육과정 편성을 시작하겠습니다.

현재 단계: Phase 1 - 킥오프

먼저 기초 자료를 확인하겠습니다:
1. 교육청 편성 지침이 있으신가요? 파일이 있다면 공유해 주세요.
2. 전년도 교육과정 편성표를 참고할 수 있나요?
3. 올해 중점적으로 추진하고 싶은 교육활동이 있으신가요?

사용자: 교육청 지침 파일은 여기 있어요. 올해는 AI 교육을 강화하고 싶어요.

교무부장(AI): 교육청 지침을 분석했습니다. AI 교육 강화를 반영하여 편성 기준안을 작성하겠습니다.

[연구부장 에이전트에게 위임]
연구부장(AI): AI 교육 관련 최신 동향과 교육청 권장 프로그램을 조사했습니다.
추천 방안: 주당 2시간 AI 리터러시 수업 + 기존 교과 연계 AI 활용 수업 ...

교무부장(AI): 연구부장의 분석을 반영하여 시간표 초안을 작성했습니다.
[시간표 초안 제시]
교감님의 검토를 받겠습니다.

교감(AI): 시간표 초안을 검토했습니다. 법정 수업시수 충족 여부를 확인했고,
교과 간 시수 배정이 적절합니다. 다만 5학년 목요일에 ...
```

### 예시 대화 2: 학교 행사 기획

```
사용자: /axys-event-planning 체육대회

교감(AI): 체육대회 기획을 시작하겠습니다!

Phase 1 - 킥오프를 진행합니다.

먼저 몇 가지 확인하겠습니다:
1. 체육대회 예정 일시는 언제인가요?
2. 작년 체육대회 프로그램을 참고할 수 있나요?
3. 올해 특별히 포함하고 싶은 프로그램이 있나요?

사용자: 5월 셋째 주 금요일이요. 작년보다 학부모 참여를 늘리고 싶어요.

교감(AI): 5월 셋째 주 금요일(5/16)로 설정하겠습니다.
학부모 참여 확대를 반영한 기획안을 수립합니다.

[각 에이전트에게 역할 배분]
- 교무부장: 수업 조정 및 학사일정 확인
- 학년부장: 학년별 참여 프로그램 계획
- 행정실장: 예산 편성 및 물품 조달
- 학생대표: 학생 참여 프로그램 기획

행정실장(AI): 체육대회 예산안을 편성했습니다.
- 용품 구입: 50만원
- 간식/음료: 30만원
- 상품: 20만원
총 100만원 규모입니다. 예산 확보 가능 여부를 확인해 주세요.
```

## Gemini CLI에서 사용하기

Gemini CLI에서는 `GEMINI.md` 파일을 참조하여 에이전트 시스템을 사용합니다.

### 기본 사용법

```bash
# 교육과정 편성 요청
gemini -p "@GEMINI.md 교육과정 편성을 도와줘"

# 학교 행사 기획
gemini -p "@GEMINI.md 체육대회 기획을 시작하고 싶어"

# 예산 편성
gemini -p "@GEMINI.md 올해 예산 편성을 도와줘"

# 특정 에이전트에게 직접 요청
gemini -p "@GEMINI.md 교무부장으로서 시간표 초안을 작성해줘"
```

### 파일을 함께 참조하기

```bash
# 교육청 지침 파일과 함께 교육과정 편성 요청
gemini -p "@GEMINI.md @교육청지침.pdf 이 지침을 반영하여 교육과정을 편성해줘"

# 전년도 예산 자료와 함께 예산 편성
gemini -p "@GEMINI.md @전년도예산.xlsx 전년도 실적을 참고하여 올해 예산을 편성해줘"
```

## FAQ (자주 묻는 질문)

### Q1. 프로그래밍을 몰라도 사용할 수 있나요?

네, 사용할 수 있습니다. 설치 과정에서 터미널에 명령어 몇 줄을 입력하는 것 외에는 프로그래밍 지식이 필요하지 않습니다. 설치 후에는 자연어(일상 대화)로 AI와 소통하면 됩니다.

### Q2. 에이전트가 실제로 업무를 처리해 주나요?

AXYS의 AI 에이전트는 **업무 보조** 역할을 합니다. 문서 초안 작성, 절차 안내, 검토 의견 제시 등을 도와주지만, 최종 결정과 공식 문서 승인은 실제 담당자가 수행해야 합니다. AI가 만든 결과물은 반드시 검토 후 사용하세요.

### Q3. 학교 정보가 외부로 유출되지 않나요?

AXYS 자체는 에이전트의 역할과 업무 절차를 정의하는 설정 파일일 뿐입니다. 실제 학교 데이터의 보안은 사용하시는 AI 플랫폼(Claude Code, Gemini CLI 등)의 보안 정책에 따릅니다. 민감한 개인정보(학생 성적, 상담 기록 등)는 AI에 입력하지 않는 것을 권장합니다.

### Q4. 우리 학교 상황에 맞게 수정할 수 있나요?

네, 완전히 커스터마이징 가능합니다. `docs/customization-guide.md` 문서를 참고하시면 에이전트 역할 수정, 새 에이전트 추가, 시나리오 변경, 학교 규모에 맞는 설정 방법을 안내하고 있습니다.

### Q5. 여러 AI 플랫폼에서 동시에 사용할 수 있나요?

네, 가능합니다. `install.sh` 실행 시 `--platform all` 옵션을 사용하면 Claude Code, Gemini CLI, Codex CLI 모든 플랫폼의 설정 파일이 동시에 생성됩니다. 각 플랫폼별로 출력 파일을 해당 프로젝트에 복사하여 사용하면 됩니다.

## 문제 해결 가이드

### 설치 스크립트가 실행되지 않는 경우

```bash
# 실행 권한을 부여합니다
chmod +x install.sh

# 다시 실행합니다
./install.sh --school "우리학교" --platform claude-code
```

### "command not found" 오류가 나는 경우

사용하려는 AI 도구가 설치되어 있는지 확인합니다:

```bash
# Claude Code 확인
claude --version

# Gemini CLI 확인
gemini --version

# Codex CLI 확인
codex --version
```

설치되어 있지 않다면 각 도구의 공식 설치 안내를 따라 설치해 주세요.

### 에이전트가 올바르게 동작하지 않는 경우

1. 설정 파일이 올바른 위치에 있는지 확인합니다:
   - Claude Code: 프로젝트 루트에 `.claude/agents/` 폴더와 `CLAUDE.md` 파일
   - Gemini CLI: 프로젝트 루트에 `GEMINI.md` 파일
   - Codex CLI: 프로젝트 루트에 `AGENTS.md` 파일

2. 설정 파일 내 학교명이 올바르게 치환되었는지 확인합니다:
   - `{{SCHOOL_NAME}}`이 아직 남아 있다면 install.sh가 정상 실행되지 않은 것입니다
   - `install.sh`를 다시 실행해 보세요

3. 에이전트 파일이 누락되었는지 확인합니다:
   - Claude Code의 경우 `.claude/agents/` 폴더에 10개의 에이전트 파일이 있어야 합니다

### 출력 파일이 생성되지 않는 경우

```bash
# output 디렉토리 확인
ls output/

# 수동으로 디렉토리 생성 후 재시도
mkdir -p output
```
